## 🖥️ مواصفات السيرفر الموصى بها

> Next.js 16 (Node SSR) + PostgreSQL + رفع صور + Web Push + Cron. الأرقام لتشغيل مريح مع هامش نمو.

### المرحلة 1 — الإطلاق (حتى ~10,000 زيارة/يوم)

| المورد | الحد الأدنى | الموصى به |
|---|---|---|
| vCPU | 2 | **4** |
| RAM | 4 GB | **8 GB** |
| تخزين | 40 GB NVMe | **80 GB NVMe** |
| نظام | Ubuntu 24.04 LTS | Ubuntu 24.04 LTS |
| Node.js | 20 LTS | **22 LTS** |
| قاعدة البيانات | Postgres 16 على نفس السيرفر | **Postgres مُدار** (Supabase/Neon) |

- **أمثلة بأسعار معقولة:** Hetzner CPX31 (4 vCPU / 8 GB ≈ $15/شهر) — الأفضل سعراً/أداءً، أو DigitalOcean / Vultr بما يعادلها. لخدمة السعودية بأقل ping: AWS Bahrain (me-south-1) أو مزوّد داخل السعودية.
- **التشغيل:** Docker أو PM2 (cluster mode بعدد الأنوية) + Nginx/Caddy كـ reverse proxy مع SSL تلقائي.
- **إلزامي أمام السيرفر:** Cloudflare (CDN + WAF + حماية DDoS — الخطة المجانية تكفي للبداية).
- **الصور:** Cloudflare R2 (لا رسوم على الـ egress) — لا تخزن على قرص السيرفر.
- **بديل بدون إدارة سيرفر (موصى به لو الفريق صغير):** Vercel (Pro ~$20/شهر) + Neon/Supabase للقاعدة + R2 للصور — CDN وcron وscaling جاهزون تلقائياً.

### المرحلة 2 — النمو (50,000+ زيارة/يوم)

- 8 vCPU / 16 GB RAM أو سيرفرين خلف load balancer
- Postgres مُدار منفصل (2 vCPU / 4 GB + connection pooling عبر PgBouncer)
- **Redis** للتواجد والجلسات والـ rate limiting (تتبع `lib/presence.ts` يكتب في القاعدة عند كل طلب — انقله لـ Redis عند التوسع)
- مراقبة موارد (Grafana / Netdata) + تنبيهات

- **الحسابات التجريبية** `demo@samel.sa` / `admin@samel.sa` كلمات مرورها ظاهرة على صفحة الدخول — احذفها أو غيّرها قبل الإطلاق بلا استثناء، وحدّث النطاق القديم `samel.sa`
